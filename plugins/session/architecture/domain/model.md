## Domain Model

### コアコンセプト

#### Spawn（新規セッション起動）

新しい tmux ウィンドウ/ペインを作成し、指定プロンプトで Claude Code セッションを起動する操作。

- **入力**: 起動プロンプト、オプションのコンテキスト
- **出力**: セッション識別子（tmux ウィンドウ名）
- **責務**: セッションの生成と初期化のみ。起動後の制御は呼び出し元が担う

#### Fork（セッション分岐）

現在のセッションから派生した新しいセッションを起動する操作。

- **入力**: 親セッション識別子、分岐プロンプト
- **出力**: 新セッション識別子
- **責務**: コンテキストを引き継いだ子セッションの生成。親セッションへの影響なし

#### Observe（出力観察）

`tmux capture-pane` を用いてセッションの現在出力を取得する操作。

- **入力**: セッション識別子
- **出力**: キャプチャされたテキスト（生データ）
- **責務**: read-only でのテキスト取得のみ。解釈・判断は行わない

### observe-loop（定期観察）

一定間隔で Observe を繰り返し、結果を呼び出し元に返却するループ。

- **動作**: `capture-pane` → 呼び出し元コールバックへ渡す → スリープ → 繰り返し
- **停止条件**: 呼び出し元からの停止指示、または対象セッションの消滅
- **状態管理なし**: ループ自体は前回との差分計算や状態保持を行わない。差分検出は呼び出し元の責務

### 依存方向

```
plugins/twl → plugins/session
```

一方向のみ許可。plugins/session は plugins/twl を参照・呼び出ししない。
