## Vision

tmux セッション操作の抽象化層を提供するプラグイン。
セッションの生成・分岐・観察を統一インターフェースで扱い、呼び出し元（plugins/twl）がセッション管理の詳細を意識しなくて済むようにする。

## Constraints

- **read-only 観察**: セッション出力の取得は `capture-pane` のみ使用。セッション内容を書き換えない
- **セッション間分離**: 各セッションは独立したコンテキストを持ち、相互に干渉しない
- **一方向依存**: plugins/twl → plugins/session の方向のみ許可。session plugin は twl plugin を知らない
- **tmux 依存の局所化**: tmux コマンドの詳細はこのプラグイン内に閉じ込める。呼び出し元は tmux を直接操作しない

## Non-Goals

- **LLM 判断**: セッション出力の解釈・パターン抽出・問題検出は plugins/twl 側の責務。session plugin は生データを返すのみ
- **状態の永続化**: セッション状態（観察結果・ログ）は呼び出し元が管理する。session plugin は状態を保持しない
- **セッション内プロセス管理**: セッション内で動作するプロセスの制御（停止・再起動等）は行わない
