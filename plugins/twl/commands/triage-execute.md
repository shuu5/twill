# 分類結果の一括処理

workflow-tech-debt-triage の分類結果（4カテゴリ）を受け取り、表示・確認・実行・サマリーを行う。

## 入力

workflow-tech-debt-triage から以下のコンテキストが渡される:
- 解決済みリスト（Issue番号・タイトル・対応spec）
- 統合候補リスト（グループ・Issue群・統合理由）
- 不適切リスト（Issue番号・タイトル・クローズ理由）
- 要継続リスト（Issue番号・タイトル・ラベル）
- 対象Issue総数

## フロー（MUST）

### Step 1: 分類結果表示

カテゴリ別テーブルで一覧表示:

```markdown
## tech-debt 棚卸し結果

**対象**: N 件の tech-debt Issue を分析

### 解決済み（spec 対応あり）— クローズ提案
| # | タイトル | ラベル | 対応 spec |
|---|---------|--------|----------|
| #8 | XXX | tech-debt/warning | user-auth |

### 統合候補 — 統合提案
| グループ | Issue | 統合理由 |
|---------|-------|---------|
| A | #12, #15 | 同一モジュール: auth |

### 不適切 — クローズ提案
| # | タイトル | クローズ理由 |
|---|---------|-------------|
| #20 | XXX | 参照先が存在しない |

### 要継続
| # | タイトル | ラベル |
|---|---------|--------|
| #25 | YYY | tech-debt/deferred-high |
```

### Step 2: ユーザー確認

カテゴリごとに一括承認を確認:

```
以下のアクションを実行しますか？

1. **解決済み N 件をクローズ** — [承認 / 個別選択 / スキップ]
2. **統合候補 N グループを統合** — [承認 / 個別選択 / スキップ]
3. **不適切 N 件をクローズ** — [承認 / 個別選択 / スキップ]

「個別選択」を選んだ場合、各 Issue について承認/除外を確認します。
除外された Issue は「要継続」として扱われます。
```

### Step 3: 一括処理実行

ユーザーが承認したアクションのみ実行。

**安全性制約（MUST）**:
- `<number>` は `gh issue list --json number` で取得した整数値のみ使用。文字列操作で生成した番号は使用禁止
- コメント文字列は `--body-file` でファイル経由で渡す（シェルメタキャラクタによるインジェクション防止）
- Issue タイトル・本文から引用するテキストは `` ` `` `$` `"` を除去してからコメントに含める

**クローズ処理**（解決済み・不適切）:
```bash
# コメントをファイル経由で渡す（インジェクション防止）
COMMENT_FILE=$(mktemp /tmp/triage-comment-XXXXXX.md)
printf '棚卸しによりクローズ: %s' "<サニタイズ済み理由>" > "$COMMENT_FILE"
gh issue comment <number> --body-file "$COMMENT_FILE"
gh issue close <number>
rm -f "$COMMENT_FILE"
```

**統合処理**:
1. グループ内で最も本文が長い（情報量の多い）Issue を代表 Issue として選定
2. 代表 Issue にコメント追加: `gh issue comment <number> --body-file <path>`
3. 他の Issue をクローズ: コメントをファイル経由（`--body-file`）で渡す

**エラー処理**: `gh issue close` が失敗した場合、エラーを記録して次の Issue に進む。失敗した Issue は完了サマリーに「処理失敗」として報告する。

### Step 4: 完了サマリー

```markdown
## 棚卸し完了

| アクション | 件数 |
|-----------|------|
| クローズ（解決済み） | N 件 |
| クローズ（不適切） | N 件 |
| 統合 | N グループ（M 件） |
| 要継続 | N 件 |
| 処理失敗 | N 件 |

合計: X 件の tech-debt Issue を処理しました。
```

---

## 禁止事項（MUST NOT）

- **ユーザー確認なしで Issue をクローズ・統合してはならない**
- **Issue 番号を推測してはならない**: controller から渡された番号のみ使用
