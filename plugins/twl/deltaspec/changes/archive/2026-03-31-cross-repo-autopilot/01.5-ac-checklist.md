## 受け入れ基準（Issue #46）

1. `autopilot-plan.sh` が `owner/repo#N` 形式の Issue 参照を解決できる
2. plan.yaml に repos セクションが定義でき、各 Issue にリポジトリ識別子が付く
3. Worker が正しいリポジトリの Pilot が作成した worktree ディレクトリで起動される
4. 異なるリポジトリの同一 Issue 番号が状態ファイルで衝突しない
5. merge-gate が正しいリポジトリの PR を操作する
6. repos セクション省略時に従来の単一リポジトリ動作が維持される（後方互換）
7. Project Board に複数リポジトリの Issue が同期される
