## Context

仕様変更（#47 フラグ廃止、#82 コンポーネント削除、#98 deps.yaml 構造変更、各種リファクタリング）後にテストが追従しておらず 70/1390 件失敗。テストはすべて `tests/scenarios/` 配下のシェルスクリプトベースのシナリオテスト。

## Goals / Non-Goals

**Goals:**

- テスト PASS 率を 100%（スキップ除く）に回復
- 旧仕様を前提としたアサーションを現行仕様に更新
- 不要になったテストファイルの削除

**Non-Goals:**

- プロダクションコードの変更
- 新規テストの追加
- テストフレームワークの変更

## Decisions

1. **テスト修正のみ、プロダクションコード変更なし**: テストが旧仕様を期待しているだけで、プロダクションコードは正しい状態にある
2. **カテゴリ別に一括修正**: 同じ原因の失敗をカテゴリ別にグループ化して効率的に修正
3. **存在しないスクリプト/コンポーネントを期待するテストは削除**: 旧仕様で存在したが現在削除済みのものをテストする意味がない
4. **カウント系テストは現行値に更新**: ファイル数・エントリ数の期待値を実際の値に合わせる

## Risks / Trade-offs

- **リスク**: テスト修正時に意図せず本来検出すべきリグレッションを見逃すテストにしてしまう可能性
  - **対策**: 各テストの意図を確認し、現行仕様で等価な検証になるよう修正
- **リスク**: カウント系テストは今後の変更で再び壊れやすい
  - **対策**: カウント系は動的取得（`loom` CLI の出力ベース）に変更可能だが、Non-Goal のため今回は期待値更新のみ
