## 受け入れ基準（Issue #110）

1. `co-autopilot --board` で Board の非 Done Issue（Todo + In Progress）を取得し plan.yaml を生成できる
2. クロスリポジトリ Issue が `repo_id#number` 形式で plan.yaml に出力される
3. Issue body の依存キーワードから Phase 分割が自動で行われる
4. `--board` と `--issues`/`--explicit` は排他的（同時指定はエラー）
5. SKILL.md の Step 0 引数解析テーブルに `--board` が追加されている
6. Board に非 Done Issue がない場合はエラーメッセージで正常終了
