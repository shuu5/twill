## 受け入れ基準（Issue #290）

1. schema v1 JSON スキーマが `plugins/session/architecture/` に定義される
2. cld-spawn が window 生成時に manifest を atomic write する
3. cld-fork が window 生成時に manifest を atomic write する
4. autopilot-launch.sh が window 生成時に manifest を atomic write する
5. kill-window / session-closed イベントで tombstone が書かれる
6. unknown field tolerance が実装される（警告のみ、エラー終了なし）
7. schema_version 不一致時は write を拒否し警告を出す
8. bats テストで atomic write と tombstone が検証される
