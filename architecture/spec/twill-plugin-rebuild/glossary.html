§4. 命名 policy 用語 (第 5 弾 dig Round 7 確定)

用語定義
canonical name各 entity の正規名 1 つ。英語 hyphen-kebab (公式 skill name 制約 verified: lowercase letters + numbers + hyphens のみ、_ 不可)
forbidden synonymentity の禁止語。twl audit --vocabulary で word boundary 厳密 detect、severity: warning
1 entity = 1 name ルール同一概念 entity に対し canonical name を 1 つだけ与える。多重 synonym は forbidden 扱いで audit
公式名衝突回避公式 Claude Code 名 (Agent/Skill/Tool/Plan/Explore) は大文字始まり + backtick、twl 内部 entity は小文字 + role-prefix
migration-stage entityStrangler Fig migration の phase 番号 (Phase 1 PoC/Phase 2 dual-stack/Phase 3 cutover/Phase 4 cleanup) は backtick + 大文字で migration-stage entity として明示分離、phase forbidden audit から除外
migration-stage 文脈許容 (Phase B 2026-05-14 追加)migration-stage は本文中の見出し・説明主語として裸表記を許容 (例: rebuild-plan.html の「Phase 1 PoC」見出し、「Phase 1 で実装」等の説明文)。vocabulary audit の exclusion_contexts で backtick あり (`Phase 1 PoC`) は migration-stage 表記として除外、backtick なし裸の phase 単独語のみ forbidden 検出対象。読みやすさ重視、spec 整合性は backtick 引用の有無で機械区別
