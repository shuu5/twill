目的

各 entity の責務詳細

administrator (L0) の責務

spawn 元
user 手動起動 (+ --continue/--resume で session 継続)。Desktop scheduled tasks で machine boot 自動起動も候補
spawn 先
pilot (phase-* 4 種) + tool-* 4 件 (限定)。spawn-tmux.sh 経由、Inv I-1 (idempotent) + I-2 (parallelism 上限) を守る
status 遷移
read only (Inv I-5)。gh project item-list で 5 min cache poll。write は絶対禁止 (user 明示介入の例外のみ)
mailbox
.mailbox/admin/inbox.jsonl のみ受信 (Inv I-4 pyramid)。worker 直接 mail 禁止、pilot 集約後に admin。flock 共有 lock で read
worktree
操作禁止 (read のみ)。git worktree コマンド実行禁止、pilot 経由
step.sh
利用しない (admin は polling cycle 6-step、step.sh framework は worker 専用)
gate hook
対象外 (admin は phase-* invocation でないため PreToolUse gate 発火しない)
audit 出力
.audit/<run-id>/admin-cycle.jsonl に全 cycle の read/judge/write 記録
受信 mail event
phase-completed / phase-failed / phase-gate-denied / spawn-completed / tool-completed (tool-sandbox-runner のみ)
送信 mail event
spawn-request / escalate-human-gate / budget-pause
escalate
3 連続 failure → ★HUMAN GATE (AskUserQuestion or Discord mail)。budget exhaustion → CronDelete + one-shot wakeup
crash recovery
heartbeat self-touch (Inv G)、session 死亡時は user 手動 --continue。CronCreate 復元 (7 日 expiry 注意)

pilot (phase-*、L1) の責務

spawn 元
administrator (admin-cycle.sh step 4)。spawn-tmux.sh wrapper 経由
spawn 先
worker のみ。worker 用 worktree を事前作成してから spawn (Inv B)
status 遷移
遷移実行専任。gh project item-edit --single-select-option-id を 1 回のみ実行。担当 phase の 1 遷移のみ責任 (phase-impl なら Refined → Implementing)
mailbox
受信: worker から .mailbox/<phase-name>/inbox.jsonl。送信: admin mailbox に要約 mail のみ (pyramid 集約)。送受信両方を行う唯一の layer
worktree
lifecycle 専任 (Inv B)。git worktree add (phase 開始時) + git worktree remove (phase 完了時)。worker は worktree を自ら作成・削除しない
step.sh
worker SKILL.md で step::run を呼び出す規約を提供。pilot 自身は step.sh 直接呼ばない
gate hook
PreToolUse gate 対象 (Inv W)。phase-gate.sh (tier 1) + twl_phase_gate_check (tier 2) が pilot invocation 時に発火、前提 status 不一致なら deny + admin notify
audit 出力
.audit/<run-id>/pilot/<name>/ 配下に phase lifecycle 全 event 記録
送信 mail event
phase-started / phase-progress / phase-completed / phase-failed / merge-gate-failed
review framework
worker 出力検証 (step post-verify 結果確認、src diff、test count 増加確認)。Inv K (Pilot 実装禁止) で src diff = pilot のみは FAIL
merge gate
phase-pr pilot のみが PR merge を実行 (Inv C 保全)。他 phase の pilot は merge 禁止
内部 subagent
1 階層内側 subagent 可 (codex CLI 等、verified 公式制約)、子 spawn 禁止

worker (worker-*、L2) の責務

spawn 元
pilot (spawn-tmux.sh)。pilot が worktree を作成してから worker を起動する順序が必須 (Inv B)
spawn 先
禁止。内部 subagent (codex CLI 等) は 1 階層のみ可 (公式 verified)。tmux new-window / claude --plugin-dir の呼び出し禁止 (cross-session pollution 防止)
status 遷移
更新禁止。pilot 経由のみ。gh project item-edit 実行は MUST NOT。worker は pilot に mail で報告し、pilot が status を遷移
mailbox
pilot の mailbox (.mailbox/<phase-name>/inbox.jsonl) にのみ書き込み。chain step ごとに step-* event を emit。flock exclusive lock で atomic write (Inv T)
worktree
自身の worktree 内での実作業のみ (cwd-guard Inv B)。pilot が作成した worktree の中で git checkout, edit, test 実行。他 Issue の worktree に触れることは MUST NOT (Inv V)
step.sh
step::run <name> <exec_fn> <verify_fn> の 4 phase lifecycle (pre-check → exec → post-verify → report) を MUST (Inv U)。self-report-only 禁止
gate hook
対象外。worker 自身が Skill invoke しないため PreToolUse gate 発火しない。gate は spawn する pilot の invocation 時に発火済み
audit 出力
.audit/<run-id>/workers/<name>/ 配下に step lifecycle 全 4 phase 記録
送信 mail event
step-started / step-completed / step-precheck-failed / step-postverify-failed
内部 subagent
1 階層内側のみ可。codex CLI 等の呼び出しは可、subagent が更に子 spawn は禁止
checkpoint
.mailbox/<worker-name>/ または per-Worker パスが必須 (Inv V)。共通 path への write は --shared フラグ明示 + audit log 必須

tool-* (3 件、独立) の責務 (第 4 弾 dig)

詳細は tool-architecture.html §2-§5 + dig-report-tools-2026-05-13.md §1-§3 参照。要約:

3 tool 構成
tool-architect (HTML spec 維持 + 軽量 PR cycle) / tool-project (GitHub Project + template + verify の 3 軽 scope) / tool-sandbox-runner (sandbox × feature matrix + Idea Issue 起票)。第 4 弾 dig で tool-utility 廃止 (commands は他 3 tool + admin inline に再分配)
spawn 元
user 明示 (slash command /twl:tool-architect <args>) または admin (tool-architect / tool-sandbox-runner のみ、ADR-040 gate 通過後)
spawn 先
spawn 禁止 (内部 subagent 可、1 階層)
status 遷移
無関係 (MUST NOT 触れる)
mailbox
tool-sandbox-runner のみ admin に tool-completed mail。他 2 tool は user に直接結果提示 (mailbox 不使用)
worktree
tool-architect は spec edit PR の独自 worktree、tool-sandbox-runner は test-target/main orphan branch の専用 worktree (verified pattern 継承)
step.sh
利用しない (status と無関係)
gate hook
対象外 (status と無関係、ADR-040 gate が SSoT)。例外: tool-architect は別系統で spec edit boundary hook (pre-tool-use-spec-write-boundary.sh) を保持し、tool-architect / user 以外の Edit/Write を deny
audit 出力
.audit/<run-id>/tools/{architect,project,sandbox-runner}/ 配下
自律 verify
tool-architect は 2 層 verify (SKILL.md MUST + PostToolUse hook verify-coverage.sh warn) で 4-state status の漏れを catch、軽量 PR cycle で changelog 漏れを worker-spec-review agent が検出 (3 回 fix loop 上限)
