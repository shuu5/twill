本 file は twill-plugin-rebuild spec の outline。draft-v2 の主要 file (README / overview / failure-analysis / adr-fate-table / invariant-fate-table / crash-failure-mode / step-verification / twl-mcp-integration / ssot-design) が先に実装され、本 file は次 session で content 実装される。

目的

administrator / pilot / worker / tool / phase の 5 entity の責務 boundary を行列形式で明示する。各 entity が何を行う / 行わない / 委譲する かを clean table で定義。

想定 outline (次 session で実装)

  1. 5 entity 概要 (administrator / pilot / worker / tool / phase の各定義)
  2. 責務 boundary matrix (table: 行 = 操作種別、列 = 5 entity、値 = MUST / MUST NOT / MAY / 委譲先)
  3. 典型シナリオでの責務分担 (Issue 起票 / refine / implement / PR review / merge / verify)
  4. cross-entity 操作の禁止事項 (pilot から admin の状態に直接 write 禁止、worker から worker 直接通信禁止、等)
  5. 失敗時の責務 (各 entity の failure mode を crash-failure-mode.html と整合)
  6. tool-* の独立性 (status 遷移と無関係、user/admin 直接 invoke のみ)
