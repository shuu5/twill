rebuild-plan — outline

status: outline · 次 session で content 実装予定
本 file は twill-plugin-rebuild spec の outline。次 session で Strangler Fig 4 phase rebuild plan の詳細を実装する。draft-v1 §11 を発展、Phase 1 PoC (#1660 sanitize) の具体手順を deep-dive。

目的

新 architecture への移行 plan を Strangler Fig 4 phase + 各 phase の verify points + rollback path で仕様化する。Phase 1 PoC は Issue #1660 (SKIP_*_REASON sanitize) を新 architecture で再実装することで proof-of-concept とする。

想定 outline (次 session で実装)

  1. Phase 1: PoC (Day 1-3) — 最小実装で 3 階層 + step verification + file mailbox + gate hook が動くことを実証
    • ★ STEP -1 (本 session 第 3 弾追加、user 要望): 既存 PreToolUse hook 全件無効化 — plugins/twl/hooks/hooks.json を hooks.json.bak に rename し、hooks.json を {"hooks":{}} (空) に置換 (or 必要最小限の安全 hook のみ残す: host-safety / commit-validate / merge-guard の 3 件)。背景: 既存 hook は新 architecture でほぼ全廃予定 (chain-runner 廃止に伴い無意味化)、error 通知が運用 noise になる。復旧: rollback 時に hooks.json.bak を rename で復元。影響: spec-review-gate / phase3-gate / refined-status-gate / pre-pr-gate / merge-gate-block 等 17 件無効化、新 phase-gate.sh (tier 1) は plugin scope に新規配置で代替
    • ★ STEP 0 (本 session 第 3 弾追加): sandbox EXP 実行 — bash experiments/run-all.sh で全 EXP-001〜018 を実行、core claim を experiment-verified に格上げ (sandbox-experiment.html §8 recommended sequence)
    • ★ STEP 1 (本 session 第 3 弾追加): twl plugin 化 — plugins/twl/.claude-plugin/plugin.json 作成、claude --plugin-dir plugins/twl で起動確認、/twl:* namespace で skill invoke 動作 EXP-010
    • ADR-043 起票 (md 本体 + 本 spec directory を supplement として link)
    • 新 helper 5 本作成 (mailbox.sh / step.sh / phase-gate.sh / spawn-tmux.sh / admin-cycle.sh、合計 ~260 lines、deletion-inventory §5 verified)
    • SKILL.md 3 個 draft (phase-impl / worker-test-ready / administrator)、plugin namespace で配置
    • PreToolUse hook 設定 (plugins/twl/hooks/hooks.json、verified schema)
    • Plugin Monitor 採用 (本 session 第 3 弾): plugins/twl/monitors/monitors.json 新規、mcp-watchdog.sh 廃止 EXP-009
    • GitHub 公式 MCP 追加: repo root .mcp.json に github server (type: "http" + X-MCP-Toolsets + --read-only)
    • PoC 完遂: Issue #1660 sanitize を新 architecture で再実装 (6 step フロー、admin polling 検知 → phase-impl spawn → worker spawn → test-scaffold + green-impl + check → mail 集約 → status Implementing → phase-pr → branch protection auto-merge)
  2. Phase 1 verify points (12 項目、本 session research 反映済)
    • VP-1: 3 階層 spawn が動く (tmux list-windows で確認)
    • VP-2: file mailbox が動く (.mailbox/.../inbox.jsonl、flock atomic、Inv T)
    • VP-3: step verification が動く (test 数増加 + RED / GREEN 確認、Inv U)
    • VP-4: gate hook が動く (Refined でない Issue に phase-impl 呼び出して deny、Inv W)
    • VP-5: admin polling が動く (CronCreate recurring=true、25 min cycle、durable は使わない #40228)
    • VP-6: context 効率化 (admin token 使用量 < 旧 su-observer)
    • VP-7: fail-fast at step level (意図的 fail で step abort)
    • VP-8: PR 完遂 (Issue #1660 auto-merge、Status=Merged)
    • VP-9 (research §7-2 追加): tier 1 + tier 2 hook が parallel 実行されることを確認 (additionalContext に両方の reason が含まれる)
    • VP-10 (research §7-2 追加): bypassPermissions mode でも tier 2 の deny が効くことを確認 (TWL_PHASE_GATE_BYPASS 未設定で実証)
    • VP-11 (research §7-2 追加): mcp-watchdog 廃止後の MCP server 'twl' not connected 発生頻度測定 (active session で 8 時間以上稼働、SIGTERM 発生 / 未発生をログ)
    • VP-12 (research §7-2 追加): twl_send_keys_safe で escape 罠を防げることを確認 (セミコロン / ハイフン / スペース / Escape を含む string を送信して破壊しない)
  3. Phase 2: dual-stack (Day 4-7) — 残り phase + 旧 freeze
    • 残り 3 phase の SKILL.md (phase-explore / phase-refine / phase-pr)
    • 残り worker 6 個の SKILL.md
    • tool-* 4 個 (rebrand from co-architect / project / utility / self-improve)
    • Project Board option 追加 (Explored / PR Reviewed の新規追加、Todo / InProgress / Done を Idea / Implementing / Merged に rename、option_id 維持)
    • Actions yaml 整理 (archive-merged.yml 新規、auto-merge.yml 改、auto-refined.yml / project-status-done.yml 削除)
    • 旧 chain-runner.sh は freeze (新規修正禁止、in-flight Issue のみ実行)
  4. Phase 3: cutover (Day 8-11) — 旧 bash 全削除
    • in-flight Issue 全完遂 or 新経路に migration
    • 旧 bash 削除 (~5000 lines、deletion-inventory.html 参照)
    • workflow-* SKILL.md 13 個 削除
    • twl MCP server 極小化 (twl-mcp-integration.html 参照)
    • chain SSoT 三重化撤廃 (ssot-design.html 参照、chain.py / chain-steps.sh / chain 削除)
    • checkpoint dir 削除
  5. Phase 4: cleanup (Day 12-14) — docs + 検証
    • README / CLAUDE.md / refs/ を新 architecture に整合
    • ADR Superseded マーク (adr-fate-table.html の Superseded chain 順序に従う、Phase 4 最後に ADR-043 を Accepted 化)
    • bats regression test 全 PASS 確認
    • 新規 bats test 追加 (file mailbox concurrent / phase-gate deny / step verification fail-fast / daemon deploy verify)
    • Project Board UI 設定 (Kanban 6-stage + Table view、Web UI 手動 or copyProjectV2)
    • Memory MCP に lesson 集約 (本 spec の deep-dive を doobidoo 保存)
    • ADR-037 番号重複正規化
  6. rollback path
    • Phase 1 PoC 失敗 → 旧 chain-runner.sh で #1660 を再実装、本 spec 修正
    • Phase 2 dual-stack 中の問題 → 該当 phase を旧 chain にフォールバック
    • Phase 3 cutover 後の問題 → git revert (commit 単位で rollback、本 spec の rebuild commit を分離するため atomic rollback 可能)
    • Phase 4 後の問題 → Phase 4 cleanup commit を revert で旧 ADR Status を復元
