§3. 10 role × concern matrix

本節は tool-architecture.html と boundary-matrix.html の参照基盤。10 role の concern 割り当てをここで一元定義する。
role location prefix description 有無 allowed-tools (代表) can_spawn 公式仕様根拠 vocabulary entry
administrator skills/administrator/ (singleton、なし) 有 全 Tool phaser-*、tool-* skills docs §4
phaser skills/phaser-*/ phaser- 有 Skill, Bash, Read, mcp__twl__* workflow-*、specialist-* (条件付き) skills docs §4
tool skills/tool-*/ tool- 有 各 tool 定義 specialist-* (条件付き) skills docs §4
workflow skills/workflow-*/ workflow- 無 (disable-model-invocation: true) Skill, Bash atomic-* (Skill() 経由) skills docs: disable-model-invocation §4

specialist agents/specialist-*.md specialist- 有 (公式必須) 各 specialist 定義 不可 (subagent spawn 禁止) sub-agents docs verbatim §4
reference refs/ref-*.md ref- 概念外 — (doc のみ) — — §4
script scripts/hooks/<event>-*.sh event-based 概念外 — (bash) — plugins-reference: hook type=command §4
hook hooks/hooks.json (entry) (registry entry) 概念外 — — plugins-reference: 27 lifecycle events §4
monitor monitors/monitors.json (entry) (registry entry) 概念外 — — plugins-reference: monitors §4

§4. vocabulary entry 6 field schema

本節は glossary.html の参照基盤。glossary.html は本節の 6 field schema に準拠して用語集を実装する。
field型必須説明例 (phaser)
canonicalstring必須唯一の正規名。英語 hyphen-kebab、lowercase + hyphens のみ (_ 不可 verified)。audit の基準名phaser
aliasesstring[]任意許容される同義語。audit では warn しない[]
forbiddenstring[]任意禁止語。twl audit --vocabulary で word boundary 検出、severity: warning[pilot, phase, controller, pilot-phase]
contextstring必須entity が属する文脈 (role level / directory / 用途)L1 role
descriptionstring (multiline)必須自由記述。日本語可。entity 参照は backtick + 英語 canonical別 session window controller...
examplesstring[]推奨実ファイル名 or 実 entity 名の例示[phaser-explore, phaser-refine, phaser-impl, phaser-pr]

6.1 audit_vocabulary() pseudo-code

def audit_vocabulary(registry_yaml, spec_dir, code_dir) -> list[dict]:
    """vocabulary table に基づき forbidden 語の使用を機械検出。"""
    glossary = registry_yaml['glossary']
    items = []
    for entity_name, entry in glossary.items():
        forbidden = entry.get('forbidden', [])
        for forbidden_word in forbidden:
            # spec/*.html、cli/twl/**/*.py、plugins/twl/**/*.sh、*.md を grep
            for file in glob_files():
                for line, content in enumerate(file):
                    # word boundary 厳密 (正規表現 \b で囲む)
                    if matches_word_boundary(content, forbidden_word):
                        # ただし context (公式名 backtick 引用、説明用 etc.) は exclude
                        if not is_excluded_context(content, forbidden_word):
                            items.append({
                                "severity": "warning",
                                "section": "vocabulary",
                                "component": f"{file}:{line}",
                                "message": f"forbidden '{forbidden_word}' for entity '{entity_name}', canonical is '{entry['canonical']}'"
                            })
    return items
