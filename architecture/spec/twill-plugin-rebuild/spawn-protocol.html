§3. spawn-tmux.sh helper 仕様 (~50 行)

3.1 引数

引数意味必須
--window-name <name>tmux window 名 (50 文字上限)必須
--skill <namespaced-name>起動 slash command (例: twl:phaser-impl)必須
--args <string>slash command に渡す引数 (例: Issue 番号)任意
--cwd <path>worker 用 worktree pathworker 時必須
--mailbox-from <name>mail 送信者識別 (sender field)必須

3.2 実装 (pseudo-bash)

#!/bin/bash
# spawn-tmux.sh — 3 階層共通 spawn helper (~50 行)
# verified: tmux new-window -d -n -c flags tmux(1)

set -euo pipefail
source "$(dirname "$0")/mailbox.sh"

main() {
  local window_name skill args cwd mailbox_from
  while [ $# -gt 0 ]; do
    case "$1" in
      --window-name) window_name="$2"; shift 2 ;;
      --skill) skill="$2"; shift 2 ;;
      --args) args="$2"; shift 2 ;;
      --cwd) cwd="$2"; shift 2 ;;
      --mailbox-from) mailbox_from="$2"; shift 2 ;;
      *) shift ;;
    esac
  done

  # 1. window 名 length check (50 文字上限、tmux 制約)
  [ "${#window_name}" -le 50 ] || _fail "window name too long: $window_name"

  # 2. 既存 window 確認 (Inv I-1 idempotent guard)
  if tmux list-windows -a -F '#{window_name}' | grep -qx "$window_name"; then
    _emit_admin "spawn-skip" "{\"window\":\"$window_name\",\"reason\":\"already exists\"}"
    return 0
  fi

  # 3. plugin 化 twl の場合、--plugin-dir で起動
  local plugin_dir="${TWILL_PLUGIN_DIR:-$CLAUDE_PROJECT_DIR/plugins/twl}"

  # 4. tmux new-window (detached、cwd 指定)
  tmux new-window \
    -d \
    -n "$window_name" \
    -c "${cwd:-$CLAUDE_PROJECT_DIR}" \
    -- "claude --plugin-dir $plugin_dir"
  # ↑ tmux man verified: -d=detached, -n=name, -c=start-directory

  # 5. slash command inject (Claude が起動して prompt ready になるまで wait)
  _wait_session_ready "$window_name" 30 || _fail "session not ready after 30s"

  # 6. inject /twl:<skill> <args>
  local cmd="/${skill} ${args:-}"
  tmux send-keys -t "$window_name" "$cmd" Enter
  # ↑ send-keys: Enter key を C-m として inject (公式 syntax)

  # 7. spawn-completed mail emit
  _emit_admin "spawn-completed" "{\"window\":\"$window_name\",\"skill\":\"$skill\"}"
}

_wait_session_ready() {
  local window="$1"; local timeout="$2"; local elapsed=0
  while [ "$elapsed" -lt "$timeout" ]; do
    if tmux list-windows -a -F '#{window_name}|#{pane_in_mode}' \
         | grep -q "^${window}|0\$"; then
      sleep 2; return 0  # ready + idle
    fi
    sleep 1; elapsed=$((elapsed+1))
  done
  return 1
}

_emit_admin() {
  local event="$1"; local detail="$2"
  mailbox_emit "$mailbox_from" "admin" "$event" "$detail"
}

_fail() { echo "spawn-tmux.sh: $*" >&2; exit 1; }
main "$@"

§5. mailbox.sh helper 仕様 (~40 行)

API:
  • mailbox_emit <from> <to> <event> <detail_json> — atomic write 1 line
  • mailbox_read <mailbox-name> [--since <line-no>] — 未読 mail を stdout に出力 (read line-no を .read-offset file に保存)
  • mailbox_drain <mailbox-name> <callback-fn> — 未読 mail を 1 件ずつ callback に渡す
  • mailbox_archive <mailbox-name> — inbox.jsonl を archive/ に移動 (Wave 完了時)
#!/bin/bash
# mailbox.sh — file mailbox helper (~40 行)
# verified pattern: session-atomic-write.sh L41-50 (flock + mktemp + mv)
# 不変条件 T: file mailbox atomic write

set -euo pipefail
MAILBOX_ROOT="${TWILL_MAILBOX_ROOT:-.mailbox}"

mailbox_emit() {
  local from="$1" to="$2" event="$3" detail="$4"
  local target_dir="$MAILBOX_ROOT/$to"
  mkdir -p "$target_dir"
  local lock_file="$target_dir/.lock"
  local inbox="$target_dir/inbox.jsonl"

  # JSON Lines 1 行 (jq -c で compact、改行なし)
  local line=$(jq -nc \
    --arg from "$from" --arg to "$to" \
    --arg event "$event" --argjson detail "$detail" \
    --arg ts "$(date -Iseconds)" \
    '{from: $from, to: $to, ts: $ts, event: $event, detail: $detail}')

  # flock atomic append (timeout 10s、fd 9)
  (
    flock -x -w 10 9 || exit 1
    echo "$line" >> "$inbox"
  ) 9>>"$lock_file"
}

mailbox_read() {
  local mailbox="$1"
  local inbox="$MAILBOX_ROOT/$mailbox/inbox.jsonl"
  local offset_file="$MAILBOX_ROOT/$mailbox/.read-offset"
  local from=$(cat "$offset_file" 2>/dev/null || echo 0)

  [ -f "$inbox" ] || return 0

  # 共有 lock で read (write は exclusive で衝突防止)
  (
    flock -s -w 10 9 || exit 1
    tail -n +$((from + 1)) "$inbox"
    local total=$(wc -l < "$inbox")
    echo "$total" > "$offset_file"
  ) 9>>"$MAILBOX_ROOT/$mailbox/.lock"
}

  "phaser-impl-1660" \

7.3 admin → pilot (spawn 経由、mailbox は使わない)

admin が pilot を起動するのは spawn-tmux.sh 経由 (mailbox 不使用)。pilot 起動後の通信は worker→pilot→admin の上方向のみ (pyramid)。

§8. mail loss 防止 + archive 戦略

  • file 永続: disk 故障以外で loss しない (verified、JSON Lines + append-only)
  • backup: .mailbox/ は .gitignore 対象、git で持たない。重要 mail は archive で残す
  • archive: Wave 完了時に mailbox_archive で .mailbox/<name>/inbox.jsonl → .mailbox/archive/inbox-<name>-<ts>.jsonl
  • retention: 7 日 retention、cron で古い archive 削除
  • audit 統合: TWL_AUDIT=1 時に audit_snapshot(.mailbox, "mailbox-pre-archive") で .audit/<run-id>/ にもコピー

§10. subagent nesting 制約 (公式 verified)

公式制約 (verified): Claude Code subagent は 1 階層内側のみ spawn 可能、子 subagent の更なる spawn は禁止。
  • worker (L2) は内部で codex CLI 等の subagent を 1 階層内に持てる
  • worker 内 subagent が更に subagent spawn は 禁止
  • これが tmux で別 main session を spawn する根拠: 3 階層 (admin / pilot / worker) は全て別 main session、subagent 入れ子では実現不可
verified source: Claude Code 公式 docs (code.claude.com/docs/en/sub-agents.md L60/L348/L737)

§11. 不変条件 (本 file が enforce)

  • Inv T (file mailbox atomic write): 全 mailbox write は flock 取得した atomic write EXP-006
  • Inv V (per-specialist checkpoint): .mailbox/<session-name>/ は per-session、共通 path 禁止 EXP-013
  • Inv I-4 (admin は pilot mail のみ受信): pyramid 集約、worker から admin 直接 mail 禁止
  • Inv I-7 (tmux send-keys 使用禁止): ただし spawn-tmux.sh の spawn 直後 1 回のみ例外として inject 許可 (skill 起動 trigger)。それ以外の通信は mailbox 経由
