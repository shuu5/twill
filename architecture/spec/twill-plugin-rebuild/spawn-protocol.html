本 file は twill-plugin-rebuild spec の outline。次 session で 3 階層 spawn 仕様 + file mailbox 通信 protocol の詳細を実装する。

目的

新 spec の 3 階層 spawn (administrator → pilot → worker) の機構と、階層型 file mailbox による通信 protocol を仕様化する。draft-v1 §5b の内容を発展 + 詳細化。

想定 outline (次 session で実装)

  1. 3 階層 spawn 機構 (tmux new-window + claude code、各層別 main session)
    • L0 administrator は user が起動
    • L1 pilot は administrator が spawn-tmux.sh で起動
    • L2 worker は pilot が spawn-tmux.sh で起動
  2. spawn-tmux.sh helper 仕様 (~30 lines、引数: session-name / skill / args / mailbox-target)
  3. file mailbox 仕様 (.mailbox/<session-name>/inbox.jsonl + flock atomic、不変条件 T)
  4. mailbox.sh helper 仕様 (~30 lines、send / recv 2 関数)
  5. message format (JSON Lines、共通 field: from / ts / event / detail / heartbeat_ts)
  6. send / recv code 例 (3 階層全方向の使用例)
  7. mail loss 防止 (file 永続、disk 故障以外で loss しない)
  8. archive 戦略 (7 日 retention、mailbox/archive/inbox-<timestamp>.jsonl)
  9. 公式 SendMessage primitive (experimental) との比較 + 採用しない理由
  10. subagent nesting 制約 (公式 verified、worker 内部 subagent は 1 階層のみ)
