拡張性 (新 file 追加 scenario)

新 file 追加時は spec-management-rules.md の R-10 decision tree で dir + sub-category が決定される。具体的な 3 scenario:

  • 新 ADR (例: ADR-0013): decisions/ 配下に MD 配置、README の decisions/ section table に entry 追加
  • 新 spec file (例: phaser-lifecycle.html): spec/ 配下 (role 責務系なので core sub)、README の spec/core table に entry + graph に node 追加 (R-1 + R-2)
  • 新 research file (例: dig-report-XXX-YYYY-MM-DD.html): research/ 配下、README の research/ table に entry 追加
