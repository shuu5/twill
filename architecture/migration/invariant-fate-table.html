不変条件 Q: budget status line 5h:N%(Xm) format 解釈の標準化

目的
tmux status line の budget 表示 5h:N%(Xm) の誤読を構造的に不能化する (2026-05-11 事故: (Xm) を残量と誤解、Pilot 緊急 kill 事故発生)。
制約
  • N% = budget 使用率 (cumulative)
  • (Xm) = 次回 5h cycle reset までの wall-clock 分 (残量ではない)
  • ScheduleWakeup / CronCreate の delaySeconds は (Xm) × 60 + 300 (cycle reset 直後 + 5 分余裕)
根拠
Issue #1577 (budget format documenting)、CLAUDE.md auto memory feedback_budget_display_misread.md。
検証方法
bats ref-invariants-structure.bats Q section 存在 + count ≥ 17 assertion。
影響範囲

  • 全 controller spawn / pilot fallback monitor の budget 解釈経路

不変条件 T: file mailbox atomic write 必須

目的
9 P0 bug の #1703 (phase-review.json cross-pollution) を構造的に不能化する。
制約
  • 全 mailbox write は flock を取得した atomic write を MUST とする
  • mailbox path は .mailbox/<session-name>/inbox.jsonl の per-session 形式
  • 共通 path への write は禁止 (横断要因 F-1 の構造的解消)
  • mailbox write には必ず {"from": "<sender>", "ts": "<iso8601>", ...} を含める (sender tracking)
根拠
9 P0 bug 分析の Bug #1703 lesson、横断要因 F-1。
検証方法
bats file-mailbox-atomic-write.bats (新規)、並列 write race condition test。
影響範囲
  • plugins/twl/scripts/mailbox.sh (新規 helper)
  • 全 worker / pilot / administrator SKILL.md (mail write 経路)

不変条件 U: Atomic skill verification (4-phase lifecycle inline 実装)

目的
9 P0 bug の #973 (RED merge silent rot)、L1873-1884 自己申告 step を構造的に不能化する。第 5 弾 dig Round 9 で旧 "Step verification framework" から rename + 改訂 (詳細: atomic-verification.html §6)。
制約 (第 5 弾 dig 改訂、Phase I 2026-05-15 確定)
  • 全 atomic skill 呼び出しは pre-check → exec → post-verify → report の 4-phase lifecycle を atomic SKILL.md 本文に inline 実装 MUST (旧 step.sh framework による外部 bash 呼び出しを廃止)
  • Step 0/2/3 (pre-check / post-verify / report) は SKILL.md に hardcode、deterministic 検証・送信 (Phase I dig Q1 確定)。Step 1 (exec) のみ LLM 自由生成 (AC file 読み込み)
  • post-verify で機械検証 (test 数増加 / RED→GREEN / src diff / Agent return value 等) を 必ず実施
  • self-report-only (record_current_step + ok のみ) は禁止
  • post-verify FAIL → atomic abort + phaser escalate (mail に failure 明記)、verdict 厳格 binary: "PASS" 以外 (FAIL/WARNING/unknown/空) は全て exit 1 (Phase I dig Q4 確定)
根拠
9 P0 bug 分析の Bug #973 lesson、L1873-1884 自己申告 step 問題。
検証方法
bats atomic-verification-post-verify.bats (旧 step-verification-post-verify.bats から rename)、test なしで atomic-green-impl 呼び出すと atomic abort することを verify。
影響範囲
  • 旧: plugins/twl/scripts/step.sh (第 3 弾 dig 案 3) — 廃止 (Phase 3 cutover)
  • 新: atomic SKILL.md 7 件 (atomic-test-scaffold / atomic-green-impl / atomic-check / atomic-pr-test / atomic-ac-verify / atomic-phase-review / atomic-pr-cycle-report)、各 SKILL.md 本文に 4-phase inline 実装
  • workflow SKILL.md は atomic を Skill で順次呼ぶ launcher (verify 詳細 inline なし)

不変条件 V: per-specialist checkpoint path (共通 path 禁止)

目的
9 P0 bug の #1703 (checkpoint cross-pollution) + #1673 (cleanup cross-Wave 破壊) を構造的に不能化する。
制約
  • 全 state / cleanup / checkpoint write は per-specialist (issue_number / session_name) を含むパス にデフォルト書き込み
  • 共通パスへの write は明示的 --shared フラグ + audit log 必須
  • cleanup スクリプトは --scope <wave-id> を必須引数とし、他 Wave のリソースに触れない
根拠
9 P0 bug 分析の Bug #1703, #1673 lesson、横断要因 F-1。
検証方法
bats per-worker-state-isolation.bats (新規)、複数 specialist simulation で他 specialist (or atomic) の state を読み書きしないことを verify。
影響範囲
  • cli/twl/src/twl/autopilot/checkpoint.py の _checkpoint_dir (per-issue 必須化、--shared 明示なしは fail)
  • plugins/twl/scripts/autopilot-cleanup.sh (--scope 必須化、または完全廃止)
